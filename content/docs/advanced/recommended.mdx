Claude Codeをもっと便利にするプラグインとスキルを紹介する。

## 公式プラグイン：Superpowers

Claude Code開発チームが提供する公式プラグイン。インストール方法は[Pluginsのインストール](/docs/plugins/installing-plugins)を参照。

### 収録Skillの一覧

| スラッシュコマンド | 内容 |
|---|---|
| `/commit` | 変更内容を解析してConventional Commits形式のコミットを自動生成 |
| `/review` | コードレビューを実行。改善点・バグ・セキュリティリスクを指摘 |
| `/debug` | 体系的なデバッグ手順でバグの根本原因を特定 |
| `/brainstorming` | 実装前に要件・設計・制約を対話形式で整理 |
| `/writing-plans` | ブレインストーミング後に詳細な実装計画を作成 |
| `/executing-plans` | 作成した実装計画を段階的に実行 |
| `/tdd` | テスト駆動開発の手順でコードを実装 |
| `/finishing-a-development-branch` | 実装完了後のPR作成・マージ手順をガイド |

## おすすめのSkill単品アイデア

自分で作ってみると便利なSkillのアイデア集。

### 作業日誌を自動作成

```markdown
---
name: daily-log
description: 今日の作業内容をまとめた日誌を作る
---

git logとgit diffで今日の変更を確認し、
logs/YYYY-MM-DD.md に作業日誌を作成する。
完了したこと・困ったこと・明日やることの3セクションで書く。
```

### PR説明文を自動生成

```markdown
---
name: pr-desc
description: プルリクエストの説明文を生成する
---

現在のブランチとmainの差分を確認し、
以下の形式でPR説明文を生成する：
- ## 変更内容（箇条書き）
- ## テスト方法
- ## スクリーンショット（UIの場合）
```

### コードの日本語コメントを追加

```markdown
---
name: add-comments
description: コードに日本語のコメントを追加する
---

指定されたファイルを読み込み、
複雑なロジックや重要な処理に日本語のコメントを追加する。
既存のコメントは変更しない。コードそのものは変更しない。
```

### デプロイ前チェック

```markdown
---
name: pre-deploy
description: デプロイ前の安全確認チェックリストを実行する
---

以下を順番に確認する：
1. テストがすべてパスしているか（npm test）
2. ビルドが成功するか（npm run build）
3. 環境変数に機密情報のハードコードがないか
4. コンソールログのデバッグ出力が残っていないか
5. 未コミットの変更が残っていないか
問題があれば報告し、すべてOKなら「デプロイ可能です」と伝える。
```

## Skillを作るときのコツ

- **動詞で始める** — 「〇〇を作る」「〇〇を確認する」と具体的に書く
- **出力形式を指定する** — 「箇条書きで」「マークダウンで」など
- **スコープを絞る** — 1つのSkillには1つの責務だけ持たせる
- **まずSkillから** — 慣れてきたらPluginにまとめる
